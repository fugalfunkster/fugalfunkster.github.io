Unbeatable Tic-Tac-Toe Game

This project was the first time I implemented a grown-up algorithm: MiniMax, a venerable, game-theory classic. Before making a move, the computer recursively calculates all possible moves, and ranks them based on their outcome. With that data, the computer simply selects the best move based on that move's possible outcomes. I made my program more performant by generating a random move whenever the computer plays first.

Meteorite Strikes on a Pierce Projection

For my last D3 project with FreeCodeCamp, I had to plot meteorites across the glode. But a simple Mercator projection would not do, oh no, I wanted to be non-conformist. So I used the Pierce quincuncial projection, a conformal projection that doesn't exagerate scale as badly as the (better known) Mercator projection. The Pierce projection's distortion occurs mostly in areas of open ocean, better preserving the shape of the continents. Also, it just looks cool.

Force Directed Network Graph

FreeCodeCamp's camper news is a posting board where campers can link to code-related news and miscellany. My network chart shows relationships between campers and their posts.

This map was my first time using the force. Force direction gives life to charts by dictating repulsive and attractive forces between nodes. When users move nodes, the other nodes respond, preserving a clean presentation.

Calculator with Closures

Though it may not look like much, this calculator helped me explore closures. Closures are dead simple: Functions have acess to their parent's scope. But closures get interesting in JS, because functions are first-class, so you can invoke a function outside of it's lexical context. Because that function still has access to it's parent's scope, it is said to have enclosed that scope. Pretty handy when you want to curry a function and reheat it later for leftovers...
